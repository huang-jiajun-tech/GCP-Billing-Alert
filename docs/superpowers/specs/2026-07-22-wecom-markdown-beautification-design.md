# GCP 费用告警企业微信 Markdown 样式美化设计

本设计文档旨在对当前系统发送给企业微信 Webhook 的 Markdown 告警消息进行美化，提升其视觉质感、专业度与在移动端的阅读体验。

## 1. 目标与设计决策

### 1.1 痛点
- 现有的 Markdown 消息包含过多的多层嵌套引用符（`>`），在企业微信客户端实际渲染时排版拥挤，视觉负担大。
- 信息层级不够清晰，颜色过于混杂。

### 1.2 解决方案：方案 A + 组合 2 (红灰极简配色)
- **卡片式布局**：移除大部分冗余的 `>` 引用符，使用水平分割线（`---`）进行段落隔离，构建精美的结构化信息卡片。
- **红灰极简配色**：
  - **警告橙红 (`warning`)**：仅用于标记超出阈值的异常核心数值（当前费用、波动涨幅）以及报警图标（🔴, 🚨, 📦）。
  - **注释灰色 (`comment`)**：用于标记所有的非异常基准数值（如历史费用）、日期范围、告警名称以及阈值配置。
  - **默认黑白**：其余正文内容。
- **信息分层**：
  1. **标题**：醒目的红色告警标题（`### 🔴 GCP 费用超标告警`）。
  2. **基本信息**：汇总告警规则、统计日期、统计周期和判定阈值。
  3. **异常详情列表**：分组显示每个超标的 Billing 账号或 Project 账号，包含当前开销、历史对比值以及变化幅度。
  4. **Top 项目明细（Billing 维度专用）**：对每一个 Billing 账号，以缩进列表形式展示当日消费前 3 的 Top 项目（以及百分比占比）。

## 2. 详细变更设计

### 2.1 [MODIFY] [scheduler.py](file:///Users/huangjiajun/Project/Billing_Alert/backend/scheduler.py)

#### 2.1.1 辅助函数 `send_webhook_alert` 兼容与美化
修改 `send_webhook_alert`，根据传入的 `message_content` 格式进行自适应解析：
- 如果内容已经是全新格式（以 `### 🔴` 开头），则直接使用。
- 如果内容是测试告警（以 `🔔` 开头），则采用全新红灰极简模板对测试通知进行卡片化包装。
- 对于其他遗留或简单文本消息，应用全新外层模板。

#### 2.1.2 Billing 维度告警格式更新
重构 Billing 级别检测触发时的文本拼接逻辑：
- 废弃带有 `>` 前缀的排版，采用 `*` 列表点、加粗和 `warning` / `comment` 色彩搭配。
- 保证测试断言所需的关键字（如 Billing 账号显示名称、超标项目 ID、涨幅比例等）全部被正确保留且格式一致。

#### 2.1.3 Project 维度告警格式更新
重构 Project 级别检测触发时的文本拼接逻辑：
- 使用统一美观的卡片结构展示项目 ID、所属客户、所属 Billing 账号以及费用、涨幅明细。
